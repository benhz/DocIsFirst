# Code Review 流程

## 为什么要 Code Review

- 提高代码质量
- 知识共享
- 发现潜在问题
- 统一代码风格

## Review 流程

1. 开发完成，创建 Pull Request
2. 指定 Reviewer（至少 1 人）
3. Reviewer 审查代码，提出意见
4. 作者修改代码
5. Reviewer 批准（Approve）
6. 合并代码

## Review 检查清单

### 功能正确性
- ✓ 是否实现了需求
- ✓ 边界情况是否处理
- ✓ 是否有明显的 bug

### 代码质量
- ✓ 命名是否清晰
- ✓ 逻辑是否简洁
- ✓ 是否有重复代码

### 性能
- ✓ 是否有性能问题（如 N+1 查询）
- ✓ 是否有内存泄漏风险

### 安全
- ✓ 是否有 SQL 注入风险
- ✓ 是否有 XSS 风险
- ✓ 敏感信息是否加密

### 测试
- ✓ 是否有单元测试
- ✓ 测试覆盖率是否足够

## 最佳实践

### 对于作者
- PR 尽量小（< 500 行）
- 添加 PR 描述
- 自己先 Review 一遍

### 对于 Reviewer
- 及时 Review（24 小时内）
- 提建设性意见
- 重要问题标注 "必须修改"
