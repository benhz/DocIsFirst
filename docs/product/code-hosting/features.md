# 功能详解

## 仓库管理

### 仓库类型
- **公开仓库**：所有用户可见，适合开源项目
- **私有仓库**：仅授权用户可访问，适合企业内部项目
- **内部仓库**：企业内部用户可见，适合企业内部协作

### 仓库组织
- **个人仓库**：个人用户创建和管理
- **组织仓库**：团队共同管理的仓库集合
- **仓库组**：按项目或部门组织仓库

### 代码浏览
- 语法高亮显示
- 文件树导航
- 历史版本浏览
- Blame 功能追踪代码作者
- 代码搜索（支持正则表达式）

## 分支管理

### 分支策略
支持多种常见的分支管理策略：
- Git Flow
- GitHub Flow
- GitLab Flow
- Trunk-Based Development

### 分支保护
- 禁止强制推送
- 要求代码评审
- 要求 CI 检查通过
- 限制可推送人员
- 要求提交签名

### 合并策略
- **Merge Commit**：保留完整历史
- **Squash Merge**：压缩为单个提交
- **Rebase Merge**：线性历史
- **Fast-Forward**：快进合并

## 代码评审

### Pull Request / Merge Request

#### 创建流程
1. 创建功能分支
2. 提交代码更改
3. 发起合并请求
4. 代码评审讨论
5. 修改并更新
6. 评审通过合并

#### 评审功能
- **行级评论**：在具体代码行添加评论
- **建议更改**：评审者可直接建议代码修改
- **评审状态**：通过/需要修改/拒绝
- **评审者指派**：自动或手动指派评审者
- **必需评审者**：设置强制评审规则

### 代码质量检查
- 自动化代码规范检查
- 单元测试覆盖率检查
- 代码重复度检测
- 安全漏洞扫描

## 权限与安全

### 权限体系

#### 仓库级别权限
- **Owner（所有者）**：完全控制权限
- **Maintainer（维护者）**：管理权限
- **Developer（开发者）**：读写权限
- **Reporter（报告者）**：只读权限
- **Guest（访客）**：有限访问权限

#### 分支级别权限
- 允许推送的用户/角色
- 允许合并的用户/角色
- 允许强制推送的用户
- 代码所有者（CODEOWNERS）

### 安全特性
- **SSH 密钥管理**：支持 RSA、Ed25519 等密钥类型
- **双因素认证（2FA）**：增强账户安全
- **访问令牌**：细粒度的 API 访问控制
- **IP 白名单**：限制访问来源
- **审计日志**：完整的操作记录
