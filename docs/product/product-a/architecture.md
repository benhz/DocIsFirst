# 产品 A 架构设计

## 整体架构

```
┌─────────────────────────────────────────────────┐
│                 前端应用层                        │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐      │
│  │ Web UI   │  │ Mobile   │  │ Desktop  │      │
│  └──────────┘  └──────────┘  └──────────┘      │
└─────────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│                 API 网关层                        │
│  认证 / 鉴权 / 限流 / 日志                        │
└─────────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│                 业务服务层                        │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐      │
│  │ 核心服务  │  │ 分析服务  │  │ 通知服务  │      │
│  └──────────┘  └──────────┘  └──────────┘      │
└─────────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│                 数据存储层                        │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐      │
│  │ MySQL    │  │ Redis    │  │ OSS      │      │
│  └──────────┘  └──────────┘  └──────────┘      │
└─────────────────────────────────────────────────┘
```

## 核心模块

### 前端应用
- **技术栈**：React + TypeScript
- **职责**：用户交互、数据展示
- **部署方式**：CDN + Nginx

### API 网关
- **技术栈**：Kong / Nginx
- **职责**：路由、认证、限流
- **关键配置**：见 [运维文档](../../operations/deployment.md)

### 核心服务
- **技术栈**：Java Spring Boot
- **职责**：业务逻辑处理
- **扩展性**：支持水平扩展

### 数据层
- **主库**：MySQL（业务数据）
- **缓存**：Redis（热数据）
- **文件存储**：OSS（附件、图片）

## 数据流

### 用户请求流程
```
用户 → CDN → API 网关 → 业务服务 → 数据库
                ↓
              缓存层
```

### 异步任务流程
```
业务服务 → 消息队列 → 后台任务 → 数据库
```

## 技术选型

| 组件 | 技术 | 原因 |
|-----|------|------|
| 前端 | React | 生态丰富、性能好 |
| 后端 | Spring Boot | 企业级、稳定 |
| 数据库 | MySQL | 成熟、可靠 |
| 缓存 | Redis | 高性能 |
