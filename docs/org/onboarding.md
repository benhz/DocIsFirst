# 新人入职指南

## 第一周：环境准备

### Day 1-2：账号和权限
- [ ] 获取公司邮箱
- [ ] 加入企业微信
- [ ] 开通 VPN
- [ ] 开通代码仓库权限
- [ ] 开通开发环境权限

### Day 3-5：文档学习
建议按以下顺序阅读：
1. [文档首页](../intro.mdx) - 了解整体
2. [系统架构](../development/architecture/system-architecture.md) - 了解架构
3. [开发规范](../development/standards/code-style.md) - 了解规范
4. [Git 工作流](../development/standards/git-workflow.md) - 了解流程

## 第二周：环境搭建

### 本地开发环境
按照 [本地环境搭建](../development/guides/local-setup.md) 搭建开发环境

### 熟悉项目
- 克隆代码
- 阅读 README
- 运行项目
- 了解项目结构

## 第三周：上手开发

### 第一个任务
- 导师分配一个简单的 Bug 或小需求
- 创建分支，编写代码
- 提交 PR，接受 Review
- 合并代码

### 参加会议
- 参加周会
- 参加需求评审
- 了解团队工作方式

## 第一个月：融入团队

### 目标
- 独立完成一个小功能
- 熟悉团队协作流程
- 了解核心业务

### 建议
- 多问问题
- 多参与讨论
- 多看代码
- 多做笔记

## 学习资源

### 内部资源
- 技术文档：本站

### 外部资源
- Java: [官方文档](https://docs.oracle.com/en/java/)

## 导师制度

每个新人都会分配一个导师：
- 解答技术问题
- Code Review
- 职业发展建议

## 常见问题

### Q: 如何联系导师？
A: 企业微信直接沟通，或在技术群提问。

### Q: 遇到问题怎么办？
A: 先自己尝试解决，查文档、搜索。如果卡住超过 30 分钟，及时求助。

### Q: 如何快速融入团队？
A: 多参与讨论，主动分享，积极协作。
