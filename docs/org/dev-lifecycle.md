# 开发生命周期

## 完整流程

```
1. 需求提出
   ↓
2. 需求评审
   ↓
3. 技术设计
   ↓
4. 开发
   ↓
5. Code Review
   ↓
6. 测试
   ↓
7. 发布
   ↓
8. 监控与反馈
```

## 各阶段详细说明

### 1. 需求提出
- PM 整理需求
- 编写需求文档
- 提交需求评审申请

### 2. 需求评审
- 参与人：PM + TL + Dev + QA
- 评审内容：需求合理性、可行性、工作量
- 输出：需求确认、排期

### 3. 技术设计
- TL 主导设计
- 编写技术设计文档
- 技术评审

**设计文档包含**：
- 系统架构
- 数据库设计
- 接口设计
- 技术选型

### 4. 开发
- 创建功能分支
- 编写代码
- 单元测试
- 自测

### 5. Code Review
- 提交 PR
- 至少 1 人 Review
- 修改意见
- 合并代码

### 6. 测试
- 部署到测试环境
- QA 执行测试用例
- 提 Bug，Dev 修复
- 回归测试

### 7. 发布
- 提交发布申请
- 审批通过
- 部署到生产
- 验证功能

### 8. 监控与反馈
- 监控关键指标
- 收集用户反馈
- 优化迭代

## 时间节点

| 阶段 | 时长 |
|-----|------|
| 需求评审 | 1 天 |
| 技术设计 | 2-3 天 |
| 开发 | 根据需求 |
| 测试 | 开发时长的 30% |
| 发布 | 半天 |

## 各阶段文档

- 需求文档：PM 负责
- 技术设计文档：TL 负责
- API 文档：Dev 负责
- 测试用例：QA 负责
