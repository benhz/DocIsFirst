# 组织架构与职责

## 研发团队组织架构

```
CTO
 ├─ 研发总监
 │   ├─ 前端 Team
 │   │   ├─ 前端 TL
 │   │   └─ 前端工程师 x N
 │   ├─ 后端 Team
 │   │   ├─ 后端 TL
 │   │   └─ 后端工程师 x N
 │   └─ 移动 Team
 ├─ 测试总监
 │   └─ 测试工程师 x N
 └─ 运维总监
     └─ 运维工程师 x N
```

## 角色与职责

### 产品经理（PM）
- 需求调研和分析
- 产品规划和设计
- 需求文档编写
- 需求评审和讲解

### 技术 Leader（TL）
- 技术方案设计
- 架构决策
- Code Review
- 团队技术指导

### 开发工程师（Dev）
- 功能开发
- 单元测试
- Bug 修复
- 技术文档

### 测试工程师（QA）
- 测试用例设计
- 功能测试
- 自动化测试
- Bug 跟踪

### 运维工程师（Ops）
- 环境维护
- 部署发布
- 监控告警
- 故障处理

## 协作流程

```
PM 提需求 → TL 技术评审 → Dev 开发 → QA 测试 → Ops 发布
```

## RACI 矩阵

| 活动 | PM | TL | Dev | QA | Ops |
|-----|----|----|-----|----|----|
| 需求评审 | R | C | C | I | I |
| 技术设计 | I | R | C | I | C |
| 开发 | I | A | R | I | I |
| 测试 | I | C | C | R | I |
| 发布 | I | C | I | C | R |

- R: Responsible（负责）
- A: Accountable（审批）
- C: Consulted（咨询）
- I: Informed（知会）
